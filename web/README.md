# Медкарта — веб-интерфейс

React + Vite + TypeScript. Патиент-фронт поверх бэкенд-API `/api/v1` (папка `../`).
Псевдоним нигде не появляется на фронте — сервер скоупит данные по сессии/мосту.

## Запуск (dev)

Нужны поднятые **Postgres + Redis** (см. `../Docker-Postgresql/docker-compose.yml`).

1. Бэкенд (из корня репозитория):
   ```
   uv run uvicorn shop.app:app --reload
   ```
   Слушает `http://localhost:8000`. Если БД пустая — применить схему и сид медицинских
   концептов (`seed_medical`), иначе формам не из чего брать category-id.

2. Фронт:
   ```
   cd web
   npm install
   npm run dev
   ```
   Vite на `http://localhost:5173`, проксирует `/api` → `:8000` (один origin, CORS не нужен).

## Флоу

- **Регистрация/вход** (`/register`, `/login`) — email + пароль (person создаётся при signup).
- При входе каркас идемпотентно вызывает `POST /me/enroll` (выпуск моста пациента —
  MVP-стенд-ин ключей) и `POST /me/session` (owner: сервер сам разворачивает мост по JWT).
- **Эпизоды** — список/создание (болезнь/травма).
- **Эпизод** — FSM-состояние и переходы, симптомы (добавить/список), загрузка документов
  (уходят в ИИ-разбор Gemini в фоне через outbox), панель полноты + красных флагов (`assess`).

## Границы MVP

- Ключи пациента — серверный стенд-ин (KeyService — заглушка); настоящая клиентская
  крипта (Ed25519/WebCrypto, owner-DEK на клиенте) — отдельной фазой закалки.
